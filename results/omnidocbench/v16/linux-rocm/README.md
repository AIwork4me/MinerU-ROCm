# Platform-standard results — v16/linux-rocm

This directory is the **OmniDocBench-ROCm platform-standard results location**
for the `linux-rocm` platform. It will hold platform-standard artifacts
(`run_summary.json`, `provenance.json`, etc.) generated by:

```bash
omnidocbench-rocm score ...
omnidocbench-rocm publish ...
```

## Current status

**Platform-standard artifacts are not yet generated.** The existing results
(located at `results/omnidocbench/v1.6/{pipeline,vlm-vllm}/`) are legacy
predictions and scores from the standalone `mineru-rocm` tooling. They contain
real, verified Full-1651 results but are not in the platform-standard schema.

To generate platform-standard artifacts, run the full OmniDocBench-ROCm
score→publish pipeline from the existing 1651 predictions. See
`docs/reproducibility.md` and the README Evaluation section for commands.

## Why this directory is empty

`community-wanted` platforms (e.g., `windows-hip`) do not need declared result
directories. This `linux-rocm/` directory is created as the target for future
platform score/publish output, but putting fake artifacts here would violate
the platform integrity contract.
